    Message Authorization

    OAuth 2.0#

    Roles#

    • Resource Owner - eg. GitHub user
    • Resource Server - eg. GitHub
    • Client - eg. Code Wars web application
    • Authorization Server -

    Authorization Code Flow#

    From Code Wars web application: GET https://github.com/login/oauth/authorize

    input:

    • client_id - GitHub identifier of Code Wars application
    • response_type - eg. code
    • redirect_uri - where the browser has to come back
    • scope - list of permissions

    output:

    • code

    Github renders a page which shows who is requesting which permissions and asks for a confirmation When accepted the browser goes back to a given redirect URI with code in the query string

    From Code Wars backend application: POST https://github.com/login/oauth/access_token

    input:

    • code

    output:

    • access_token
